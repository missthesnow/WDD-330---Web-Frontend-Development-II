Objects

Object Literals

Self-contained set of related values and functions acting as a collection of named properties that map to any JavaScript value (i.e., strings, numbers, booleans, arrays, functions, etc.) If a property’s value is a function, it is known as a method. They can be employed to encapsulate code that can be reused throughout a program. They can also inherit properties from other objects. Objects are often used to keep any related information and functionality together in the same place.

Creating Objects

To create an object literal, simply enter a pair of curly braces.

const spiderman = {};

It’s also possible to create an object using a constructor function.

const spiderman = new Object();

Accessing Properties

You can access the properties of an object using the dot notation and bracket notation (the property is represented by a string inside square brackets, so needs to be placed inside single or double quotation marks)

Dot notation is much more common, but bracket notation has the advantage of being the only way to access nonstandard property and method names that don’t follow the variable naming rules.
It also lets you evaluate an expression and use it as the property key

Computed Properties

The ability to create objects with computed property keys was introduced in ES6. This means that JavaScript code can be placed inside square brackets and the property key will be the return value of that code. The value of a property has always been allowed to be a JavaScript expression.

Calling Methods

To call an object’s method we can also use dot or bracket notation. Calling a method is the same as invoking a function, so parentheses need to be placed after the method name:

superman.fly()
<< 'Up, up and away!'
superman['fly']()
<< 'Up, up and away!'

Checking if Properties or Methods Exist

The in Operator

'city' in superman;
<< false

Check if property or method doesn’t return undefined :

superman.city !== undefined;
<< false

Finding all the Properties of an Object

We can loop through all of an object’s properties and methods by using a for in loop. For example, to log all the properties of the superman object to the console, we could use:

for(const key in superman) {
    console.log(key + ": " + superman[key]);
}
<< "name: Superman"
<< "real name: Clark Kent"
<< "height: 75"
<< "weight: 235"
<< "hero: true"
<< "villain: false"
<< "allies: Batman,Supergirl,Superboy"
<< "fly: function (){
    console.log(\"Up, up and away!\");
}"

Properties Can Be:

Added:

superman.city = 'Metropolis';
<< 'Metropolis'

Changed:

superman['real name'] = 'Kal-El';
<< 'Kal-El'

Removed:

delete superman.fly
<< true

Nested Objects

Objects can be nested like so:

const jla = {
    superman: { realName: 'Clark Kent' },
    batman: { realName: 'Bruce Wayne' },
    wonderWoman: { realName: 'Diana Prince" },
    flash: { realName: 'Barry Allen' },
    aquaman: { realName: 'Arthur Curry' },
}

Objects as Parameters to Functions

Note that objects are copied by reference!

Namespacing

Naming collisions occur when the same variable or function name is used for different purposes by code sharing the same scope. This problem an be encountered when using code librares or when working in teams.
Solution: Use the object literal pattern to create a namespace for groups of related functions.
This is done by creating an object literal that serves as the namespace, then adding any values as properties of that object, and any functions as methods.

JSON

JavaScript Object Notation, a lightweight data-storage format that is used for data serialization and configuration. Used to exchange information between web services.
JSON is a string representation of the object literal notation.

Key differences:

    Property names must be double-quoted
    Functions are not permitted values

Permitted Values:

    double-quoted strings
    numbers
    true
    false
    null
    arrays
    objects

The Math Object

A built-in object that has several properties representing mathematical constants, as well as methods that carry out a number of common mathematical operations. All the properties and methods of the Math object are immutable.

The RegExp Object

A regular expression is a pattern that can be used to search strings for matches to the pattern. A common use is 'find and replace' type operations.
Example, say you were looking for any word ending in 'ing', you could use the regular expression

/[a-zA-Z]+ing$/

Metacharacters

    . matches any character, except line breaks
    \w matches any word character, and is equivalent to [A-Za-z0-9_]
    \W matches any non-word character, and is equivalent to [\^A-Za-z0-9_]
    \d matches any digit character, and is equivalent to [0-9]
    \D matches any non-digit character, and is equivalent to [^0-9]
    \s matches any whitespace character, and is equivalent to [ \t\r\n\f]
    \S matches any non-whitespace character, and is equivalent to [^ \t\r\n\f]

Modifiers (can be escaped using a backslash)

    ? makes the preceding token in the regular expression optional
    * matches one or more occurrences of the preceding token
    + matches one or more occurrences of the preceding token
    {n} matchesnoccurrences of the preceding token
    {n,} matches at leastnoccurrences of the pattern
    {,m} matches at mostmoccurrences of the preceding token
    {n,m} matches at leastnand at mostmoccurrences of the preceding token
    ^ marks the position immediately before the first character in the string
    $ marks the position immediately after the last character in the string
