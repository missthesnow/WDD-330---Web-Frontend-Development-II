Events

Event Listeners

They are like setting a notification to alert you when something happens. Instead of the program having to constantly check to see if an event has occurred, the event listener will let it know when the event happens, and the program can then respond appropriately. This allows the program to continue with other tasks while it waits for the event to happen.

The Event Object

Whenever an event handler is triggered by an event, the callback function is called. This function is automatically passed an event object as a parameter that contains information about the event.

Mouse, Keyboard, and Touch Events

An Example

function doSomething(event){
    console.log(event.type);
}

Removing Event Listeners

An event listener can be removed using the removeEventListener() method

const onceParagraph = document.getElementById('once');
onceParagraph.addEventListener('click', remove);
function remove(event) {
    console.log('Enjoy this while it lasts!');
    nceParagraph.style.backgroundColor = 'pink';
    onceParagraph.removeEventListener('click',remove);
}

Stopping Default Behavior

Some elements have default behavior associated with certain events.
preventDefault() is a method of the event object that can be used inside the callback function to stop the default behavior happening

const brokenLink = document.getElementById('broken');

brokenLink.addEventListener('click',(event) => {
    event.preventDefault();
    console.log('Broken Link!');
});

Event Propagation

When you click on an element, you are actually clicking on all the elements it’s nested inside of. For Example:

<ul id='list'>
    <li>one</li>
    <li>two</li>
    <li>three</li>
</ul>

If you click on one of the <li> elements, you’re also clicking on the <ul> , <body> and <html> elements. An event is said to propagate as it moves from one element to another.
