CSS3 Transforms and Transitions

Transforms

The transform property lets you translate, rotate, scale, and/or skew any element on the page. Some of these effects were possible using previously existing CSS features (i.e. translating with relative and absolute positioning) CSS3 gives far more control over many more aspects of the element’s appearance.

Possible transformations include:

    Translation
    Scaling
    Rotation
    Skew

Transitions

Transitions allow the values of CSS properties to change over time, essentially providing simple animations. If the browser lacks support for transitions, the change will be immediate instead of gradual, which is fine and accessible. In addition, native CSS transitions generally require less client-side processing than JavaScript animations, so they’ll usually appear smoother.

This is is an irreplacable benefit when it comes to mobile devices with limited computing power.

Transition properties include:

    transition-duration
    transition-timing-function
    transition-delay
    transition shorthand

    Possible uses for transitions include:

      Multiple Transitions
      Animations

    Canvas, SVG, and Drag and Drop

    Canvas

    HTML5 has a Canvas API with which we can draw any number of things through JavaScript. This can improve the performance of a website by avoiding the need to download images off the network. With canvas, we can draw shapes and lines, arcs and text, gradients and patterns. Canvas also gives us the power to manipulate pixels in images and even video.

    The following platforms support the Canvas API:

      Chrome 4+
      Firefox 2+
      Opera 9.6+
      Safari 3.1+
      iOS 3.2+
      Internet Explorer 9.0+
      Android 3.0+

    Canvas must be inserted as its own element, which can then be grabbed from the DOM after giving the canvas an ID like so:

    <canvas id="myCanvas" class="myCanvas">
        Sorry! Your browser doesn’t support Canvas.
    </canvas>

    SVG

    SVG stands for Scalable Vector Graphics, a specific file format that allowsthe description of vector graphics using XML. A major selling point of vector graphics in general is that, unlike many other image types (bitmaps, i.e. GIF, JPEG, PNG, and TIFF), vector images preserve their quality regardless of scaling. We can be used SVG in may similar ways to the canvas.

    Basic SVG is supported on the following platforms:

      Chrome 4+
      Safari 3.2+
      Firefox 3+
      Opera 9.6+
      iOS 3.2+
      Internet Explorer 9.0+
      Android browser 4.4+

    Canvas versus SVG?

    Depends! Canvas allows for pixel manipulation, a downside however though - it operates in what’s known as 'immediate' mode. This means that if you ever want to add more to the canvas, you’re unable to simply add to what’s already there. Every time you finish drawing a shape, the canvas no longer has access to that shape, because it won’t persist as an object that you can modify - you'd have to start from scratch. Canvas does allow you to save the images you create to a PNG or JPEG file though.

    Canvas is also much faster than SVG because there's no access it on the DOM.

    What you draw to SVG is accessible via the DOM, because its mode is 'retained' mode, meaning that the structure of the image is preserved in the XML document that describes it. SVG also has, at this time, a more complete set of tools to help you work with it.

    However, since SVG is a file format—rather than a set of methods that allows you to dynamically draw on a surface—you can’t manipulate SVG images the way you can manipulate pixels on canvas. It would have been impossible, for example, to use SVG to convert our color video to black and white as we did with canvas.

    Drag and Drop

    The Drag and Drop API allows us to specify that certain elements are draggable, and then specify what should happen when these draggable elements are dragged over or dropped onto other elements on the page.

    The Drag and Drop API is supported on the following platforms:

      All recent versions of Chrome, Firefox, and Safari
      Partial support in Internet Explorer 7.0+ (IE10 doesn’t allow setting the drag image, but supports all other features)
      Opera 12+
