Declaring Variables

"let" allows the variable to be changed later as needed

"const" means the variable cannot be reassigned to another value

Scope

Block Scope - "const" and "let" are block scoped - their value only exists inside the block they are declared in.

Global Scope - Declared outside block; applicable to entire program.

(Personal Note: The Romani bot largely makes use of global variables, but according to the text, it would be better practice to keep these as few as possible in order to avoid potentially overwriting variables. Local Scope - Global variables can be overwritten in the block. (See Also: Block Scope)

Variable Naming

Variable names are case sensitive, so "Variable" "variable" and "VARIABLE" are 3 seperate variables.

Variables with multiple names can be written in camel case or with underscores.
i.e., thisIsAVariable versus this_is_a_variable

Booleans

Truthy values are values in JavaScript that are true.
Falsy values consist of 9 values and are always false. They are:

    "" // double quoted empty string literal
    '' // single quoted empty string literal
    `` // empty template literal
    0
    -0 // considered different to 0 by JavaScript!
    NaN
    false
    null
    undefined

Arrays, Logic, and Loops

Arrays

An array literal (an ordered list of values) is written with a pair of square brackets

Add a value to an array like so:

array[0] = 'Value';

Note that the array value starts at zero (like everything else in computing).

Remove an array value like so:

delete avengers[3];

Logic

if statements

if (condition) {
// code to run if condition is true
}

else statements

if (condition) {
// code to run if condition is true
} else {
// code to run if condition is false
}

Ternary Operator

condition ? (//code to run if condition is true) : (//code to run if condition is false)

Switch Statements

if (number === 4) {
console.log('You rolled a four');
} else if (number === 5) {
console.log('You rolled a five');
} else if(number === 6){
console.log('You rolled a six');
} else {
console.log('You rolled a number less than four');
}

Functions

Function Declarations

function hello(){
console.log('Hello World!');
}

Invoking a Function

hello();
<< 'Hello world!'

Return Values

function howdy(){
return 'Howdy World!';
}

Function invokations can be assigned to a variable like so:

const message = howdy();
<< 'Howdy World!'

Default Parameters

function hello(name='World') {
console.log(`Hello ${name}!);
}

Where 'World' is the default. The default can be overridden like so:

hello('Universe');
<< 'Hello Universe!'

Arrow Functions

const square = x => x*x;

Why use arrow functions?

    Much less verbose than normal function declarations
    Single parameters do not need to be put into parentheses
    The body of the function does not need to be placed inside a block if it's only one line
    The return keyword isn't required if the return statement is the only statement in the body of the function
    They do not bind their own value of this to the function

Callbacks

function sing(song) {
console.log(`I'm singing along to ${song}`);
}

sing('Let It Go')
<< 'I'm singing along to Let It Go'

The function can be made more flexible by adding a callback parameter like so:

function sing(song,callback) {
console.log(`I'm singing along to ${song}.`);
callback();
}
