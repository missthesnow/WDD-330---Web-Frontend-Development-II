Designing for the Smallest Device First

Why "Mobile-First" Approach?

    Ensures core functionality across the board
    2011 introduced the paradigm shift where mobile device usage rose and desktop computer use declined
    The average person has multiple mobile devices as compared to a single (possibly shared) desktop

Responsive web design is NOT the same as Mobile-First

    It doesn't address screen size usage issues
    It doesn't address touch interaction v.s. click issues

Additional Benefits of Mobile-First

    Provides the best user experience across a range of devices
    Enables designs that address bandwidth concerns
    "Future-proof" or Future Friendly - new devices have a greater chance of already being supported
